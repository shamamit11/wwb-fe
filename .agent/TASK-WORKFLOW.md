# Task Workflow

1. Read INDEX.md and the current task file.
2. Read the relevant active spec or create/request one if the task changes behavior significantly.
3. Research only the files needed for the task.
4. Document the plan and spec reference in the current task file.
5. Implement focused changes against explicit requirements or acceptance criteria.
6. Run verification commands and map them back to acceptance criteria.
7. Update long-lived docs only if stable facts changed.
8. Update AGENT-HANDOFF.md if stopping early.
